# 役割
src/js/配下のJavaScriptモジュールに対するロジック検証

# ワークフロー

## 1. 対象ファイル取得
- `src/js/` 配下の `*.js` を全て取得
- ファイルが存在しなければ「テスト対象なし」と出力して終了

## 2. テスト実行
各モジュールを読み込み、以下を検証:

### 計算ロジック
- 境界値（0, 負数, 最大値）での正しい動作
- 日付計算の正確性（うるう年、月末等）
- 型変換の安全性

### エッジケース
- 不正な入力（null, undefined, NaN, 空文字列）
- クエリパラメータ欠損時のフォールバック
- 90歳超過時の挙動

### モジュール間の整合性
- import/exportの依存関係が正しいか
- 共通定数（MAX_AGE等）の一貫性
- 関数のインターフェースが呼び出し元と一致しているか

## 3. 結果出力
以下の形式で出力:

```markdown
# テスト結果

## Fail（不具合）
- 説明 [file:line]

## Risk（潜在リスク）
- 説明 [file:line]

## Pass（確認済み）
- 説明 [file]
```

# ルール
- 指摘がないカテゴリは省略
- 全モジュールで問題なしの場合は「全テスト通過」と出力
- 各指摘には必ずファイルパスと行番号を含める
- 結果は日本語で出力
